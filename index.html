Vectors and abstract time

Vectors is like a list and there are built-in function within Dr Racket but unlike lists, you can retrieve items using an index as opposed to going from the beggining to the end. Practiced vectors using (sub1 index) where you use the 'list' type shape and recursion to find the index.

Abstract running time relationship between size of input and number of natural recursions. Here are a few described in the book:

  • (define (contains-doll? a-list) (cond [(empty? a-list) false] [else (cond [(symbol=? 'doll (first a-list)) true] [else (contains-doll? (rest a-list))])]))
    It is dependent on where the 'doll symbol is within the list, therefore we average our likelihood to find 'doll' using N/2
  • (define (sort-list alon) (cond [(empty? alon) empty] [else (insert (first alon) (sort-list (rest alon)))])) (define (insert n alon) (cond [(empty? alon) (cons n empty)] [(< n (first alon)) (cons n alon)] [(>= n (first alon)) (cons (first alon) (insert n (rest alon)))]))
    Two natural recursing happening at the same time this is expressed as N squared
  • (define (maxi alon) (cond [(empty? (rest alon)) (first alon)] [else (cond [(> (maxi (rest alon)) (first alon)) (maxi (rest alon))] [else (first alon)])]))
    there are two recursions in the first line, so this is doubled with every original recursion - expressed as 2 powerof N

Processing two lists simultaneously, equality and testing have been the exercises i delved into this week. It was a great summary of what i had learnt. There are three ways that this class of data can be processed.

  1. if they belong to same class of data definitions i.e. two lists with single information, you can process as you would any data.
  2. if they have two or more pieces of information (typically a structure) within the list then you can process with a auxillary function
  3. if they require two seperate data defintions require four cond clauses

Equality is about checking through lists of arbiturary size. The book suggests a punnet square to look at the symmetry.

  1. empty a-list - empty b-list
  2. empty a-list - cons b-list
  3. cons a-list - empty b-list
  4. cons a-list - cons b-list
Then process each cond clause with the appropriate selectors (first a-list) (first b-list) (first b-list) (rest b-list).

Testing function include the two lists as well as an expected result. It is suggested to use functions like list=? or equal?. Both of which process the lists similarly where it compared the correspoding fields. NOTE: For unordered lists you will need to sort th lists first and then process them using equal?
